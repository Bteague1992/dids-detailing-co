# SEO Location Landing Pages Guide

This guide explains how to manage the SEO-oriented location landing pages at `/mobile-car-detailing-[citySlug]-nc`.

## Overview

The site includes two types of city pages:

1. **Human-friendly pages** (`/service-area/[citySlug]`): User-facing city pages with friendly URLs
2. **SEO landing pages** (`/mobile-car-detailing-[citySlug]-nc`): SEO-optimized pages with keyword-rich URLs

Both page types use the same components and city data, ensuring consistency while serving different purposes.

## File Locations

- **SEO Pages**: `app/mobile-car-detailing-[citySlug]-nc/page.tsx`
- **SEO Content Generator**: `src/lib/seo-content.ts`
- **City Data Config**: `src/config/business.ts`
- **Sitemap**: `app/sitemap.ts`

## How to Edit the Intro Copy Template

The SEO intro text is generated by the `generateCitySeoIntro()` function in `src/lib/seo-content.ts`.

### Current Template

```typescript
export function generateCitySeoIntro(city: ServiceCity): string {
  return `Looking for professional mobile car detailing in ${city.name}, NC? ${city.description} We provide convenient mobile auto detailing services for cars, trucks, and SUVs throughout ${city.name}. Our mobile service means we come to your home or workplace - no need to leave your location. Simply text us to book your detail today.`;
}
```

### To Customize

1. Open `src/lib/seo-content.ts`
2. Modify the `generateCitySeoIntro()` function
3. The function receives a `city` object with:
   - `city.name`: City name (e.g., "Hickory")
   - `city.slug`: URL slug (e.g., "hickory")
   - `city.description`: Description from config

### Example Customization

```typescript
export function generateCitySeoIntro(city: ServiceCity): string {
  return `Professional mobile car detailing in ${city.name}, NC. ${city.description} We specialize in mobile auto detailing for cars, trucks, and SUVs. Our convenient service comes to your home or office in ${city.name}. Text us today to schedule your detail!`;
}
```

## How to Add New Cities

All cities are managed in `src/config/business.ts`. Adding a new city automatically creates both page types.

### Steps

1. Open `src/config/business.ts`
2. Add a new city object to the `serviceAreaCities` array:

```typescript
{
  slug: "new-city",           // URL-friendly identifier (lowercase, hyphens)
  name: "New City",           // Display name
  description: "Brief description of service in this city.",
}
```

3. Save the file
4. Rebuild the site: `npm run build`

The new city will automatically get:
- A page at `/service-area/new-city`
- An SEO page at `/mobile-car-detailing-new-city-nc`
- Inclusion in the sitemap
- Footer links (if within the first 4-6 cities)

### Example

```typescript
serviceAreaCities: [
  // ... existing cities
  {
    slug: "maiden",
    name: "Maiden",
    description: "Professional mobile car detailing in Maiden. Quality service at your location.",
  },
]
```

## How to Toggle Between Rendering vs Redirecting SEO Pages

Currently, SEO pages render full content. To redirect them instead (e.g., redirect to the human-friendly page), modify `app/mobile-car-detailing-[citySlug]-nc/page.tsx`.

### Option 1: Redirect to Human-Friendly Page

Replace the entire page component with:

```typescript
import { redirect } from "next/navigation";
import { businessConfig } from "@/src/config/business";

export async function generateStaticParams() {
  return businessConfig.serviceAreaCities.map((city) => ({
    citySlug: city.slug,
  }));
}

export default async function CitySeoPage({
  params,
}: {
  params: Promise<{ citySlug: string }>;
}) {
  const { citySlug } = await params;
  const city = businessConfig.serviceAreaCities.find(
    (c) => c.slug === citySlug
  );

  if (!city) {
    notFound();
  }

  redirect(`/service-area/${citySlug}`);
}
```

### Option 2: Conditional Rendering (Environment Variable)

Add a config flag and conditionally render or redirect:

1. Add to `src/config/site.ts`:

```typescript
export const siteConfig = {
  // ... existing config
  seoPagesEnabled: process.env.NEXT_PUBLIC_SEO_PAGES_ENABLED !== "false",
} as const;
```

2. In `app/mobile-car-detailing-[citySlug]-nc/page.tsx`:

```typescript
import { siteConfig } from "@/src/config/site";

export default async function CitySeoPage({ params }: { params: Promise<{ citySlug: string }> }) {
  const { citySlug } = await params;
  const city = businessConfig.serviceAreaCities.find((c) => c.slug === citySlug);

  if (!city) {
    notFound();
  }

  if (!siteConfig.seoPagesEnabled) {
    redirect(`/service-area/${citySlug}`);
  }

  // ... rest of component
}
```

3. Set environment variable:
   - To enable: `NEXT_PUBLIC_SEO_PAGES_ENABLED=true` (or omit, defaults to true)
   - To disable: `NEXT_PUBLIC_SEO_PAGES_ENABLED=false`

### Option 3: Feature Flag in Config

Add a simple boolean flag in `src/config/site.ts`:

```typescript
export const siteConfig = {
  // ... existing config
  redirectSeoPages: false, // Set to true to redirect instead of render
} as const;
```

Then check this flag in the page component.

## SEO Page Features

### Metadata

- **Title**: "Mobile Car Detailing in [City], NC | Dad's Mobile Detailing Co."
- **Description**: Auto-generated, 140-160 characters, includes key phrases
- **Canonical**: Points to the SEO page URL itself

### Content

- **H1**: "Mobile Car Detailing in [City], NC"
- **SEO Intro**: 2-3 sentences with keywords (mobile detailing, cars/trucks/SUVs, convenience, text booking)
- **Internal Links**: 
  - Links to `/service-area/[citySlug]` (human-friendly page)
  - Links to `/packages`
  - Links to `/service-area` hub

### Components Reused

- `PackagesGrid` - Same package cards
- `HowItWorks` - Same process explanation
- `FAQList` - Same FAQ content
- All CTAs use the same SMS booking function

## Sitemap

The sitemap (`app/sitemap.ts`) automatically includes:
- All static pages
- All city pages (`/service-area/[citySlug]`)
- All SEO pages (`/mobile-car-detailing-[citySlug]-nc`)

SEO pages have a priority of 0.9 (higher than regular city pages at 0.8).

## Navigation

SEO pages are **NOT** in the main navigation. They're discoverable via:
- Footer links (first 4 cities shown)
- Internal links from `/service-area` page
- Sitemap
- Direct URL access

## Best Practices

1. **Keep content unique**: The SEO intro should be unique per city while maintaining keyword density
2. **Update descriptions**: Keep city descriptions in `business.ts` fresh and relevant
3. **Monitor performance**: Track which SEO pages perform best in search results
4. **Internal linking**: Maintain links between SEO pages and human-friendly pages for SEO value

## Troubleshooting

### Pages not generating

- Check that city slugs are valid (lowercase, hyphens only)
- Verify `generateStaticParams()` includes all cities
- Run `npm run build` to see static generation output

### TypeScript errors

- Ensure all imports use `@/` alias correctly
- Check that `ServiceCity` type matches your city objects

### Sitemap not updating

- Clear `.next` cache: `rm -rf .next`
- Rebuild: `npm run build`
- Check `app/sitemap.ts` includes all city arrays
